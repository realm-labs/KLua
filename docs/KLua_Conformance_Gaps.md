# KLua Conformance Gaps

This note tracks known Lua 5.5 gaps that are too broad to treat as incidental test failures. Keep it source-backed and concise; move items out when they are implemented and covered by tests.

This list is evidence and scope tracking, not a FIFO task queue or a commit map. Select related gaps into a named campaign in `docs/KLua_Codex_Goal.md`, with an owning subsystem or Lua source helper chain, affected entry points, a case matrix, verification, and an exit condition. Group fixes and tests for the same semantic rule; do not create a separate work package or commit for every probe.

## Base Library

- `next`, `pairs`, and `ipairs` are source-backed across table and primitive states, canonical integer/float control keys, raw traversal, iterator light-function identity, exact default tuples, the fourth closing value for `pairs`, `__pairs` lookup/call/result adjustment and yielding, `__index` traversal, wrapped integer controls, termination, ignored extras, and diagnostics. `getmetatable`/`setmetatable` and `rawequal`/`rawget`/`rawset`/`rawlen` cover table and userdata validation, raw identity and access, canonical numeric keys, nil/NaN boundaries, false-valued metatable protection, source-ordered checks, ignored extras, exact arity, and errors.
- `assert`, `error`, `pcall`, and `xpcall` are source-backed across success/failure tuples, non-string object identity, nil normalization, C-`int` error-level narrowing and locations, callable targets, target yield/resume, non-yieldable message handlers, handler failures and result adjustment, handler-before-close ordering, repeated handling of replacement close errors, normalized close arguments, validation, ignored extras, exact arity, API protected calls, and coroutine resume/close/wrap propagation.
- `collectgarbage` is source-backed across operation and parameter C strings, defaults, running/mode controls, all six encoded parameters, C-`int` narrowing, rounding and saturation, result shapes, validation, ignored extras, stable reopen identity/state persistence, and per-state isolation. HotSpot exposes no embeddable collector equivalent: `collect`/`step` request `System.gc`, `count` reports JVM heap usage, and running/mode/parameter controls are logical compatibility state. Lua `__gc` finalization remains unimplemented and requires a separate VM lifecycle campaign.
- `warn` is source-backed across the enabled `luaL_newstate` runtime default, string/numeric conversion, per-part C-string boundaries, multipart continuation and on/off/unknown control routing, validation atomicity, zero-result shape, stable reopen identity/state persistence, output-consumer updates, and per-state isolation. KLua adapts each completed warning line to a configured `Consumer<String>` instead of exposing the C handler's fragment writes to stderr; standalone-style default suppression remains available through `warn("@off")` as in `lua.c`.
- `print` is source-backed across left-to-right conversion and output, partial conversion failures, separator placement, primitive and metamethod formatting, numeric metamethod results, non-yieldable calls, embedded NUL length preservation, zero arguments/results, stable reopen identity, and output-consumer updates. Successful lines and failed-call prefixes are each adapted to one `Consumer<String>` event because the host output contract does not expose the C macros' fragment/newline distinction.
- All 23 `luaopen_base` functions preserve light-C-function-equivalent identity across reopen. Stateless and yieldable helpers use stable global host functions; collector/print/warning controls and conditional `dofile`/`loadfile` use weakly state-associated stable functions without retaining their `LuaState` keys. `_G` identity, `_VERSION`, safe/unsafe conditional availability, and existing entry-point behavior remain covered.

## Package Library

- Native C module loading is intentionally unavailable in the pure Kotlin runtime. `package.loadlib` exposes the source fallback's exact C-string-checked `nil, message, "absent"` result, and the C/C-root searcher slots can report `package.cpath` misses or dynamic-loading failures but cannot create native loaders. `package.searchpath` follows Lua's C-string substitution, empty-template diagnostics, and host openability probe; Java invalid-path and security failures are treated as unreadable candidates. `require` and direct built-in searcher calls use source-compatible C-string module-name boundaries for cache/preload keys, path and symbol construction, C-root detection, and diagnostics while preserving the complete original Lua string as the loader's first argument. Package initialization uses the official platform defaults, versioned-first environment selection, first-`;;` default insertion, Windows executable-directory substitution, and registry `LUA_NOENV` with a JVM config default. Package loaded/preload storage and `debug.getregistry` now share one state-owned registry identity across re-open and global replacement.
- `package.searchpath` and the Lua/C/C-root path searchers are source-backed across name/separator substitution before expanded-path splitting, including semicolons introduced by module names; empty/repeated templates, first-open selection, exact miss diagnostics/arity, loader-data filenames, C-string/numeric boundaries, path initialization, and platform openability. Java input-stream opening substitutes for C `fopen`, and native C loader creation remains unavailable.
- `require`, the preload searcher, and custom searcher iteration are source-backed across C-string cache/search names versus original loader arguments, captured registry identities, truthy/false/nil cache states, raw searcher traversal, string-convertible numeric diagnostics, loader arguments/data/first-result handling, loader-controlled recursion, default-true caching, errors, and exact fresh/cached arity. Native C searchers preserve their diagnostics and symbol-selection protocol but cannot produce native loaders.

## Table Library

- `table.create` and `table.pack` are source-backed across zero, negative, oversized, and full-width allocation hints; optional and ignored arguments; numeric-string conversion; ordered diagnostics; nil holes; every Lua value kind; exact `n` metadata; result arity; and independent result identity. Allocation hints are validated but intentionally do not force eager JVM allocation.
- `table.insert` and `table.remove` are source-backed across ordinary, sparse, and table-like sequences; wrapped full-width and negative lengths/positions; numeric coercion; exact result arity; indexed metamethod routing; source-ordered validation and callbacks; and retained partial mutations after later shift failures.
- `table.concat` is source-backed across ordinary and table-like receivers, raw byte values and separators, Lua numeric conversion, custom and full-width ranges, exact result arity, indexed metamethod routing, and ordered diagnostics. Result allocation remains limited by JVM indexed strings.
- `table.unpack` is source-backed across ordinary, sparse, string, and table-like receivers; default, empty, and full-width ranges; nil/result arity; span overflow; length/index routing; coercion; and diagnostics. KLua's JVM result list has no direct equivalent of Lua's dynamically available C-stack slots, so very large otherwise-valid result counts remain capacity-model dependent.

## IO Library

- The official Lua 5.5 `linit.c` standard-library list includes `LUA_IOLIBNAME`; KLua now has an initial pure-Kotlin `io` library with basic file handles, source-backed line-iterator ownership including fourth-result early-exit closing for `io.lines(filename)`, common file read formats, Lua-style numeric write formatting and subtype-preserving capped numeric read scanning, partial-write byte accounting, C-string-bounded open/default/lines/process routing, C-string and signed-range-safe seek boundaries, source-ordered close finalization and result shapes, and observable `setvbuf` no/full/line modes, plus non-closing standard handles and source-routed stream-backed `io.popen`. Explicit full and line buffering uses a deterministic 1024-byte JVM default, drains on capacity/newline as applicable and before flush/seek/close, and rejects nonnegative sizes outside the JVM's 32-bit array range with a failure tuple; handles otherwise begin in direct-write mode instead of inheriting platform C stdio defaults. JVM process results cannot distinguish POSIX signal termination, and Windows C pipe text-mode translation is unavailable; broader `liolib.c` edge-case parity still needs conformance hardening.
- `io.read`, `file:read`, and the shared line-iterator read path are source-backed across line/all/numeric/count formats, raw NUL/high bytes, LF and retained CR behavior, zero-count EOF probes, partial final chunks, exact result arity, Lua-integer conversion, incremental validation and consumption, invalid numeric prefixes, subtype-preserving 200-byte numeric scanning, locale decimal points, and direct/default-handle diagnostics. Unlike Lua's platform `size_t` allocation attempt, KLua rejects negative or greater-than-`Int.MAX_VALUE` character counts as out of range before JVM allocation.
- `io.write`, `file:write`, `io.flush`, and `file:flush` are source-backed across raw byte strings, integer and round-trip-safe float formatting, multiple-value ordering, returned-handle identity, empty writes, ignored flush extras, direct/default routing, exact success and failure result shapes, partial byte counts, buffered drain sequencing, direction errors, and source-ordered conversion failures. JVM stream failures use the available Java message and numeric placeholder code rather than platform C errno.
- Regular `io.open`, `io.tmpfile`, `io.input`, `io.output`, `io.close`, and `io.type` behavior is source-backed across the complete `r`/`w`/`a`, optional-plus, and repeated-binary mode grammar; update-mode creation/truncation/append rules; C-string filenames and modes; numeric filenames; default identity and atomic replacement; standard, closed, temporary, and non-file handles; ignored extras; result arity; and ordered diagnostics. JVM filesystem failures retain Java messages and numeric placeholder codes instead of platform C errno.
- `file:seek` and `file:setvbuf` are source-backed across option-before-offset/size validation, defaults, C-string and invalid options, signed/full-width values, all origins, failed-position preservation, stream-handle failures, all buffer modes, capacity boundaries, ignored extras, exact result shapes, and drain-before-seek/reconfigure/flush/close behavior including partial drain failures. KLua exposes deterministic JVM buffering with a 1024-byte default and rejects full/line sizes outside the nonnegative 32-bit array range; native C `setvbuf` allocation, negative-size, and platform seek-width behavior can differ.
- `file:lines`, default `io.lines`, and named `io.lines` are source-backed across exact one-versus-four constructor results, caller-owned versus automatic closing, generic-for early-exit close values, empty and multiple formats, exact iterator result arity, delayed incremental validation, zero-count and ordinary EOF, read errors, repeated closed calls, 250-format limits, and source-ordered diagnostics. As in Lua, an `"a"` format succeeds with an empty string at EOF, so that iterator remains live until explicitly or generically closed.
- `io.popen` process streams are source-backed across host-specific mode grammar, C-string command/mode boundaries, default and ignored arguments, read/write direction, buffering and flush/close sequencing, exact open and successful/nonzero close result arity, automatic-close suppression, and closed/type behavior. JVM process status cannot distinguish POSIX signal termination from a numeric exit code, and Windows/JVM pipe text translation follows Java streams rather than the C runtime's `_popen` text mode.

## OS Library

- KLua's pure-JVM `os` library covers the Lua 5.5 entry points, source-backed filesystem argument/result ordering including host-conditional rename replacement, real command-shell probing with distinct launch-error versus completed-exit tuples, source-backed `os.date` string boundaries, ordered `os.time` field normalization/write-back including explicit DST-state hints, exact pre-conversion `os.difftime`, process-CPU `os.clock`, C-string-bounded environment lookup, and embedder-safe `os.exit` signaling. The date formatter deliberately exposes the Lua source's C99/POSIX conversion and `E`/`O` modifier set on every JVM host instead of switching to the reduced C89 set or Windows `#` modifiers. It maps locale data through Java, uses stable custom `%c`/`%x`/`%X` layouts, has no native `strftime` 250-byte per-conversion cap or zero-length failure signal, and inherits the JVM `Instant`/timezone database range. Time normalization uses Java timezone transitions to emulate `mktime`; zones without a matching transition and the C runtime's installation-specific historical normalization can still differ. `os.setlocale` accepts only Java-installed locale names but maps every Lua category to Java's one process-wide default locale because the JVM has no C-style independent category state. `os.clock` falls back from JVM process CPU duration to current-thread CPU when process accounting is unavailable, environment names rejected or denied by Java produce nil, and `os.exit` delegates actual state closing/process termination to the configured handler. JVM filesystem/process exceptions do not expose portable C errno values, so operation failure codes remain numeric placeholders; JVM process results also cannot reproduce POSIX wait-status signal metadata, and broader host/platform parity remains incomplete.
- `os.remove`, `os.rename`, and `os.tmpname` are source-backed across C-string and numeric filenames, source-ordered conversion, file and empty-directory removal, same-directory moves, host-conditional replacement, missing/invalid path failures, ignored extras, unique non-created temporary names, exact success/failure arity, and suffix-free diagnostics. Java filesystem messages and numeric placeholder codes remain host-specific substitutes for C errno.
- `os.execute` and `os.exit` are source-backed across one-result nil shell probes, C-string and numeric commands, completed zero/nonzero exits, missing commands, JVM launch failures, exact three-result command tuples, boolean/integer/full-width exit statuses, close-state truthiness, ignored extras, ordered diagnostics, configured handler calls, and non-returning exit propagation. Java reports launch exceptions with placeholder code 1 and exposes only numeric process exits, so KLua cannot reproduce C `errno` values or POSIX signal termination metadata; actual process termination/state closing remains delegated to the embedder's exit handler.
- `os.date`, `os.time`, and `os.difftime` are source-backed across C-string table-format detection and length-aware formatting, UTC/local results, configured conversion/modifier validation, ordered field and metamethod access, exact `yday`-before-`wday` normalized write-back, defaults, DST hints and transition normalization, sentinel and full-width boundaries, overflow-free differences, exact result arity, ignored extras, and diagnostics. KLua uses Java locale data and the JVM timezone database, stable custom `%c`/`%x`/`%X` layouts, and the configured C99 format set on every host; it therefore does not reproduce installation-specific `strftime` spellings/size failures, C89/Windows option reductions, historical `mktime` normalization, or native `time_t` ranges.
- `os.setlocale`, `os.getenv`, and `os.clock` are source-backed across C-string/numeric conversion, locale/category validation order, query/change/reset results, environment presence/absence and Java-invalid names, exact result arity, ignored extras, process-CPU continuity, and diagnostics. Java provides one process-wide default locale instead of independent C categories, only Java-installed locale names, an immutable process environment, and process CPU metrics that can fall back to current-thread accounting; locale spellings and unavailable/security-denied environment behavior can therefore differ from a native C installation.

## Strings And UTF-8

- `string.len`, `sub`, `reverse`, `rep`, `byte`, and `char` are source-backed across the complete raw-byte domain, relative and full-width extreme byte positions, empty and reversed intervals, exact result arity, numeric coercion, repetition sizing, and ordered diagnostics. Their only retained size limitation is the general JVM indexed-array/string bound described below.
- `string.find` plain search is source-backed across raw embedded NUL/control/high/UTF-8 bytes, empty and overlapping needles, relative and full-width extreme starts, byte-index/result projection, parser bypasses, coercion, and diagnostics. Its automatic plain-search decision uses Lua's exact special-byte set, including the source-specific literal treatment of unmatched `)` and scanning beyond embedded NULs.
- KLua strings are still JVM-string-backed rather than byte-string-faithful in all standard-library paths. This leaves Lua 5.5 byte-level behavior incomplete for broader byte-sensitive standard-library cases. `string.format` preserves raw literal/value bytes, applies `%s` width and precision by byte count, retains unmodified embedded NULs, rejects modified NUL-bearing strings, quotes raw bytes with source escape rules, truncates C-format diagnostics at NUL, and implements the complete numeric conversion grammar and integer/decimal/C99-hexadecimal projection with exact binary rounding, width, precision, alternate-form, special-value, coercion, and diagnostic-order coverage. Its deterministic ties-to-even and inf/NaN spellings do not track a host process that changes the C floating-point rounding mode or uses installation-specific `printf` spellings; the official C89 hexadecimal fallback also rejects modifiers that KLua's C99-compatible path supports. `string.pack`, `packsize`, and `unpack` now use C-string-bounded formats and source-compatible integer widths, endian projection, alignment, `X`, overflow/sign extension, position clipping, result shape, and diagnostic order across every explicit width from 1 through 16. Their `f`/`d`/`n` paths use IEEE binary32/binary64 layouts, exact default-rounding narrowing, signed-zero and normal/subnormal/infinity boundaries, quiet-NaN payload/sign preservation, endian reversal, alignment, source numeric-string coercion, and source-ordered errors. Their `c[n]`/`s[n]`/`z` paths preserve raw payload bytes, fixed padding, all 1-through-16 unsigned length-prefix layouts, wide-prefix validation, zero termination, alignment, positions, result shape, and diagnostic ordering. Their native-size model is deliberately stable LP64 with an eight-byte double Lua number (`long`/`size_t`/Lua integer/Lua number eight bytes, maximum native alignment eight) because the JVM has no C ABI; Lua builds with another C data or number model expose different `l`/`L`/`T`/`n` sizes, and KLua does not follow a mutable host floating-point rounding mode or non-IEEE representation. Aggregate packed strings remain limited by JVM array/string indexing instead of the larger theoretical C `size_t` range. `string.lower`/`string.upper` transform raw bytes with exact ASCII C/UTF-8-locale behavior while preserving every other byte and length under any selected Java locale. String pattern classes classify one unsigned raw byte through the same explicit ASCII policy across ordinary classes, complements, bracket sets/ranges, frontiers, captures, and every search/replacement consumer; quantified-item backtracking, capture-state rollback/result projection, structural-error reachability, once-only empty-match progression, and `%b` nesting/first-close behavior for arbitrary raw delimiters are source-backed across `find`, `match`, `gmatch`, and `gsub`. `string.gsub` follows source argument ordering and preserves raw direct/function/table replacement values, capture selection, nil/false results, and non-yieldable callback boundaries. Non-ASCII mappings and classifications from installation-specific single-byte C locale tables remain unavailable because the JVM exposes Unicode mappings rather than a 256-byte `ctype` table. The raw-byte `utf8.offset`, `utf8.len`, `utf8.codepoint`, and `utf8.codes` paths are likewise source-backed for representable malformed layouts; the remaining gap is general byte-string fidelity rather than those covered entry-point families.
- `string.dump`, binary `load`, and binary `loadfile` are source-backed across Lua-function validation, strip truthiness and debug metadata, raw package bytes, direct and reader inputs, text/binary/default mode gates, C-string mode/chunk/file controls, filename-before-mode validation, source names, environment replacement including explicit nil, file and standard-input routing, malformed/truncated packages, ignored extras, and exact success/failure arity. They round-trip through KLua's internal `KLua` bytecode package format using the stdlib raw-byte helpers; that format is deliberately not Lua's native host-dependent binary-chunk layout, so dumps are not portable to the reference runtime or other Lua implementations. General Lua byte-string fidelity also remains incomplete outside the covered paths.

## Math Library

- The integer/IEEE boundary family for `math.abs`, `math.floor`, `math.ceil`, `math.fmod`, `math.modf`, `math.tointeger`, and `math.ult` is source-backed across signed limits, subtype conversion, signed zero, infinities, NaN, unsigned ordering, arity, and argument errors. `math.min` and `math.max` follow the Lua 5.5 `lua_compare` less-than path across first-value retention on ties and NaN, exact mixed integer/float boundaries, unsigned raw-string ordering, table/userdata metamethod selection and argument order, original result identity/subtype, first-error termination, and non-yieldable native-library comparison callbacks. `math.atan` covers `atan2` quadrants for signed zero, infinities, and NaN plus the optional second argument, while `math.log` follows the source's natural, base-2, base-10, and general-base branches across IEEE domain edges; its dedicated base-2 path returns exact integral logarithms for every representable power of two, including subnormals. The direct `math.asin`, `math.acos`, `math.sqrt`, `math.sin`, `math.cos`, `math.tan`, and `math.exp` wrappers are covered across signed zero, domain or representative finite boundaries, underflow/overflow where applicable, subnormals, maximum finite values, infinities, NaN, coercion, result projection, and argument errors. `math.deg` and `math.rad` preserve the source's multiply-by-precomputed-ratio ordering, avoiding spurious intermediate overflow and covering signed zero, large finite values, infinities, NaN, and round trips. `math.frexp` and `math.ldexp` cover signed-zero and special-value projection, normal/subnormal endpoints, exact reconstruction, subtype/arity, underflow/overflow, and the source's Lua-integer-to-C-`int` exponent narrowing. The default library surface exactly matches `luaopen_math`, excludes compatibility-only names, preserves `pi`/`huge`/integer-limit subtypes, reuses light-function identity across reopen, and gives each reopened table independent random closures/state. The Lua 5.5 xoshiro random state and interval projection are implemented. The remaining explicit limitation is host `libm` approximation parity for nontrivial transcendental results across JVM and C-library installations.
