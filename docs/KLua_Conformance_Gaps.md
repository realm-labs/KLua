# KLua Conformance Gaps

This note tracks known Lua 5.5 gaps that are too broad to treat as incidental test failures. Keep it source-backed and concise; move items out when they are implemented and covered by tests.

This list is evidence and scope tracking, not a FIFO task queue or a commit map. Select related gaps into a named campaign in `docs/KLua_Codex_Goal.md`, with an owning subsystem or Lua source helper chain, affected entry points, a case matrix, verification, and an exit condition. Group fixes and tests for the same semantic rule; do not create a separate work package or commit for every probe.

## M20 Closure Matrix

The optimized runtime passed the complete Gradle suite and the canonical JDK 17 performance/allocation screen after the representation changes. The matrix below classifies the remaining differences; no row contains an unowned v1 conformance blocker.

Non-callable Lua invocation errors project source-backed local/upvalue/global/field/method/constant/iterator/metamethod call-site names, C-string-truncate diagnostic names, preserve named host-userdata types, and omit the suffix when the callee origin is not inferable; native protected targets without a Lua calling opcode retain the plain form.
- `collectgarbage` is source-backed across operation and parameter C strings, defaults, running/mode controls, all six encoded parameters, C-`int` narrowing, rounding and saturation, result shapes, validation, ignored extras, stable reopen identity/state persistence, and per-state isolation. Explicit `collect` drives a deterministic Lua-value tracer and protected `__gc` registry covering assignment-time discovery, table/host-userdata identity, global/registry/API/active/suspended-coroutine roots, closure/table/metatable/uservalue edges, all four C-string `__mode` states, weak-value clearing, weak-key ephemeron convergence, atomic finalizer-target marking/reconvergence, newest-first scheduling, resurrection, explicit re-arming, hook suppression, warnings, and at-most-once removal. Logical `step` cycles perform atomic separation then at most one finalizer per later slice, automatic table/closure allocation debt uses the state `stepsize`, stop/restart and explicit stopped stepping are connected, nested finalizer calls preserve the source's nil versus `param == -1` results and validation order, and automatic failures use the lifecycle warning policy. - `print` is source-backed across left-to-right conversion and output, partial conversion failures, separator placement, primitive and metamethod formatting, numeric metamethod results, non-yieldable calls, embedded NUL length preservation, zero arguments/results, stable reopen identity, and output-consumer updates. - All 23 `luaopen_base` functions preserve light-C-function-equivalent identity across reopen. Stateless and yieldable helpers use stable global host functions; collector/print/warning controls and conditional `dofile`/`loadfile` use weakly state-associated stable functions without retaining their `LuaState` keys. `_G` identity, `_VERSION`, safe/unsafe conditional availability, and existing entry-point behavior remain covered.

## Package Library

- Native C module loading is intentionally unavailable in the pure Kotlin runtime. `package.loadlib` exposes the source fallback's exact C-string-checked `nil, message, "absent"` result, and the C/C-root searcher slots can report `package.cpath` misses or dynamic-loading failures but cannot create native loaders. `package.searchpath` follows Lua's C-string substitution, empty-template diagnostics, and host openability probe; Java invalid-path and security failures are treated as unreadable candidates. `require` and direct built-in searcher calls use source-compatible C-string module-name boundaries for cache/preload keys, path and symbol construction, C-root detection, and diagnostics while preserving the complete original Lua string as the loader's first argument. Package initialization uses the official platform defaults, versioned-first environment selection, first-`;;` default insertion, Windows executable-directory substitution, and registry `LUA_NOENV` with a JVM config default. Package `_CLIBS`, loaded, and preload storage plus `debug.getregistry` share one state-owned registry identity across re-open and global replacement; ordinary `__index`/`__newindex` access participates in named-subtable reuse/creation, and each present path environment variable independently consults `LUA_NOENV` as `setpath` does.
- `package.searchpath` and the Lua/C/C-root path searchers are source-backed across name/separator substitution before expanded-path splitting, including semicolons introduced by module names; empty/repeated templates, first-open selection, exact miss diagnostics/arity, loader-data filenames, C-string/numeric boundaries, path initialization, and platform openability. Java input-stream opening substitutes for C `fopen`, and native C loader creation remains unavailable.
- `require`, the preload searcher, and custom searcher iteration are source-backed across C-string cache/search names versus original loader arguments, captured registry identities, truthy/false/nil cache states, raw searcher traversal, string-convertible numeric diagnostics, loader arguments/data/first-result handling, loader-controlled recursion, default-true caching, errors, and exact fresh/cached arity. Native C searchers preserve their diagnostics and symbol-selection protocol but cannot produce native loaders.

## IO Library

JVM process results cannot distinguish POSIX signal termination, and Windows C pipe text-mode translation is unavailable; broader `liolib.c` edge-case parity still needs conformance hardening.
- `io.read`, `file:read`, and the shared line-iterator read path are source-backed across line/all/numeric/count formats, raw NUL/high bytes, LF and retained CR behavior, zero-count EOF probes, partial final chunks, exact result arity, Lua-integer conversion, incremental validation and consumption, invalid numeric prefixes, subtype-preserving 200-byte numeric scanning, locale decimal points, and direct/default-handle diagnostics. - `io.write`, `file:write`, `io.flush`, and `file:flush` are source-backed across raw byte strings, integer and round-trip-safe float formatting, multiple-value ordering, returned-handle identity, empty writes, ignored flush extras, direct/default routing, exact success and failure result shapes, partial byte counts, buffered drain sequencing, direction errors, and source-ordered conversion failures. - Regular `io.open`, `io.tmpfile`, `io.input`, `io.output`, `io.close`, and `io.type` behavior is source-backed across the complete `r`/`w`/`a`, optional-plus, and repeated-binary mode grammar; update-mode creation/truncation/append rules; C-string filenames and modes; numeric filenames; default identity and atomic replacement; standard, closed, temporary, and non-file handles; ignored extras; result arity; and ordered diagnostics. ## OS Library

- KLua's pure-JVM `os` library covers the Lua 5.5 entry points, source-backed filesystem argument/result ordering including host-conditional rename replacement, real command-shell probing with distinct launch-error versus completed-exit tuples, source-backed `os.date` string boundaries, ordered `os.time` field normalization/write-back including explicit DST-state hints, exact pre-conversion `os.difftime`, process-CPU `os.clock`, C-string-bounded environment lookup, and embedder-safe `os.exit` signaling. The date formatter deliberately exposes the Lua source's C99/POSIX conversion and `E`/`O` modifier set on every JVM host instead of switching to the reduced C89 set or Windows `#` modifiers. It maps locale data through Java, uses stable custom `%c`/`%x`/`%X` layouts, has no native `strftime` 250-byte per-conversion cap or zero-length failure signal, and inherits the JVM `Instant`/timezone database range. Time normalization uses Java timezone transitions to emulate `mktime`; zones without a matching transition and the C runtime's installation-specific historical normalization can still differ. `os.setlocale` accepts only Java-installed locale names but maps every Lua category to Java's one process-wide default locale because the JVM has no C-style independent category state. `os.clock` falls back from JVM process CPU duration to current-thread CPU when process accounting is unavailable, environment names rejected or denied by Java produce nil, and `os.exit` delegates actual state closing/process termination to the configured handler. JVM filesystem/process exceptions do not expose portable C errno values, so operation failure codes remain numeric placeholders; JVM process results also cannot reproduce POSIX wait-status signal metadata, and broader host/platform parity remains incomplete.
- `os.remove`, `os.rename`, and `os.tmpname` are source-backed across C-string and numeric filenames, source-ordered conversion, file and empty-directory removal, same-directory moves, host-conditional replacement, missing/invalid path failures, ignored extras, unique non-created temporary names, exact success/failure arity, and suffix-free diagnostics. - `os.execute` and `os.exit` are source-backed across one-result nil shell probes, C-string and numeric commands, completed zero/nonzero exits, missing commands, JVM launch failures, exact three-result command tuples, boolean/integer/full-width exit statuses, close-state truthiness, ignored extras, ordered diagnostics, configured handler calls, and non-returning exit propagation. - `os.date`, `os.time`, and `os.difftime` are source-backed across C-string table-format detection and length-aware formatting, UTC/local results, configured conversion/modifier validation, ordered field and metamethod access, exact `yday`-before-`wday` normalized write-back, defaults, DST hints and transition normalization, sentinel and full-width boundaries, overflow-free differences, exact result arity, ignored extras, and diagnostics. KLua uses Java locale data and the JVM timezone database, stable custom `%c`/`%x`/`%X` layouts, and the configured C99 format set on every host; it therefore does not reproduce installation-specific `strftime` spellings/size failures, C89/Windows option reductions, historical `mktime` normalization, or native `time_t` ranges.

Java provides one process-wide default locale instead of independent C categories, only Java-installed locale names, an immutable process environment, and process CPU metrics that can fall back to current-thread accounting; locale spellings and unavailable/security-denied environment behavior can therefore differ from a native C installation.

## Strings And UTF-8

- `string.len`, `sub`, `reverse`, `rep`, `byte`, and `char` are source-backed across the complete raw-byte domain, relative and full-width extreme byte positions, empty and reversed intervals, exact result arity, numeric coercion, repetition sizing, and ordered diagnostics. Their only retained size limitation is the general JVM indexed-array/string bound described below.
- `string.find` plain search is source-backed across raw embedded NUL/control/high/UTF-8 bytes, empty and overlapping needles, relative and full-width extreme starts, byte-index/result projection, parser bypasses, coercion, and diagnostics. Its automatic plain-search decision uses Lua's exact special-byte set, including the source-specific literal treatment of unmatched `)` and scanning beyond embedded NULs.
- KLua strings now own immutable bytes; VM equality, hashing, table keys, ordering, length, concatenation, lexer/compiler constants, bytecode, exact-byte API access, and byte-sensitive string/UTF-8 entry points no longer depend on JVM text storage. `string.format` preserves raw literal/value bytes, applies `%s` width and precision by byte count, retains unmodified embedded NULs, rejects modified NUL-bearing strings, quotes raw bytes with source escape rules, truncates C-format diagnostics at NUL, and implements the complete numeric conversion grammar and integer/decimal/C99-hexadecimal projection with exact binary rounding, width, precision, alternate-form, special-value, coercion, and diagnostic-order coverage. Its deterministic ties-to-even and inf/NaN spellings do not track a host process that changes the C floating-point rounding mode or uses installation-specific `printf` spellings; the official C89 hexadecimal fallback also rejects modifiers that KLua's C99-compatible path supports. `string.pack`, `packsize`, and `unpack` now use C-string-bounded formats and source-compatible integer widths, endian projection, alignment, `X`, overflow/sign extension, position clipping, result shape, and diagnostic order across every explicit width from 1 through 16. Their `f`/`d`/`n` paths use IEEE binary32/binary64 layouts, exact default-rounding narrowing, signed-zero and normal/subnormal/infinity boundaries, quiet-NaN payload/sign preservation, endian reversal, alignment, source numeric-string coercion, and source-ordered errors. Their `c[n]`/`s[n]`/`z` paths preserve raw payload bytes, fixed padding, all 1-through-16 unsigned length-prefix layouts, wide-prefix validation, zero termination, alignment, positions, result shape, and diagnostic ordering. Their native-size model is deliberately stable LP64 with an eight-byte double Lua number (`long`/`size_t`/Lua integer/Lua number eight bytes, maximum native alignment eight) because the JVM has no C ABI; Lua builds with another C data or number model expose different `l`/`L`/`T`/`n` sizes, and KLua does not follow a mutable host floating-point rounding mode or non-IEEE representation. Aggregate packed strings remain limited by JVM array/string indexing instead of the larger theoretical C `size_t` range. `string.lower`/`string.upper` transform raw bytes with exact ASCII C/UTF-8-locale behavior while preserving every other byte and length under any selected Java locale. String pattern classes classify one unsigned raw byte through the same explicit ASCII policy across ordinary classes, complements, bracket sets/ranges, frontiers, captures, and every search/replacement consumer; quantified-item backtracking, capture-state rollback/result projection, structural-error reachability, once-only empty-match progression, and `%b` nesting/first-close behavior for arbitrary raw delimiters are source-backed across `find`, `match`, `gmatch`, and `gsub`. `string.gsub` follows source argument ordering and preserves raw direct/function/table replacement values, capture selection, nil/false results, and non-yieldable callback boundaries. Non-ASCII mappings and classifications from installation-specific single-byte C locale tables remain unavailable because the JVM exposes Unicode mappings rather than a 256-byte `ctype` table. The raw-byte `utf8.offset`, `utf8.len`, `utf8.codepoint`, and `utf8.codes` paths are likewise source-backed for malformed layouts.

### Byte-Oriented Storage Requirements

The closure audit follows `lstring.c:luaS_newlstr`, `luaS_eqstr`, and the short/long hash paths; `lapi.c:lua_pushlstring` and `lua_tolstring`; `llex.c:read_string`, `read_long_string`, and `luaX_newstring`; `lobject.c:luaO_str2num` and `luaO_tostring`; the byte-buffer entry points in `lstrlib.c`; and the decoder paths in `lutf8lib.c`. The focused matrix covers embedded NUL, all high-byte roles, valid and malformed UTF-8, byte length/indexing, concatenation, numeric/string conversion, raw equality/table keys, and host-to-core-to-host transit. No Lua 5.5 executable is configured locally, so the expected differential side is derived directly from those owning source functions.

The byte-oriented representation package closes against these requirements together:

- `LuaString` owns immutable bytes and a byte count. Equality, hashing, table keys, ordering, VM length, and concatenation operate directly on those bytes; no semantic operation may depend on UTF-16 code-unit count or require a byte-to-`String`-to-byte round trip.
- Lexer short and long literals, compiler constants, bytecode dump/load, IO reads, string builders, and standard-library producers construct strings through explicit byte paths. Source and host text use an explicit UTF-8 projection, while arbitrary byte inputs remain uninterpreted.
- Java/Kotlin text access is a boundary materialization, not storage. The public APIs need an unambiguous byte-oriented push/read path for arbitrary Lua strings; existing `String` methods retain their documented text projection, including a reversible policy for malformed bytes, without allowing wrapper spelling to change Lua equality or table lookup.
- Numeric coercion continues to require whole-string byte consumption and therefore rejects embedded NUL or nonnumeric suffix bytes. APIs that the C source treats as zero-terminated keep their existing C-string-bounded view; length-aware APIs preserve embedded NUL.
- `string` and `utf8` functions consume the same immutable bytes. UTF-8 validation never normalizes, replaces, or drops malformed input, and raw substring/capture/replacement results share or copy byte ranges without changing content.
- Short/common-string interning, if profiling justifies it, keys only by byte content and length. Hash caching belongs to the immutable byte object; interning must not leak into public identity semantics or the serialized bytecode contract.
